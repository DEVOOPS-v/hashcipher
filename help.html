Where is my data stored?
Your data is never stored on any server. Hascipher operates entirely within your browser's local RAM. Once you refresh or close the tab, all session history and keys are permanently wiped.
What is the Session Key?
It acts as a unique cryptographic salt for your encryption. By sharing this key with your recipient, they can decipher the hash. Without the exact key, the data remains a secure, unreadable string.
Is this 100% secure?
We use industry-standard client-side encryption. Because the decryption never happens on a server, it is safe from "man-in-the-middle" attacks on our end. Only those with the key can see your data.

System Operations Logic

Follow these logical steps to use the Hascipher Engine effectively.

Step 1: Generate a Session Key
First, click "Generate New Key". This creates a unique identifier for your current session. You cannot encrypt or decipher without an active key. Copy this key and send it securely to your recipient.
Step 2: Encrypting Data
Enter your message in the Encryption Input and hit Encrypt. The system uses your Session Key to obfuscate the text. Copy the resulting hash and send it to your recipient.
Step 3: Deciphering Logic
Paste the hash into the Decipher Input. When you click Decipher, a Security Modal will appear. You must paste the exact session key used during encryption to unlock and view the original text.
Note: If the key does not match the hash perfectly, the system will return an error to ensure no data is leaked.

Troubleshooting

"Invalid Key" Error
This happens if the session key is missing a character or has an extra space. Double-check that the key you are pasting is identical to the one used to encrypt.
Results Disappeared
If you refreshed the page, the browser memory was cleared. Because we do not store data, you will need to re-enter your key and hash to see the results again.
